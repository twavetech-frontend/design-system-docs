# 디자인 시스템 가이드

안녕하세요! 우리만의 멋진 디자인 시스템 문서를 만드는 곳입니다.

## Design Principles

아임인 디자인시스템은 다음 4가지 원칙을 기준으로 디자인하고 의사결정합니다.

### 1. 사용자의 익숙한 경험을 존중합니다. <span style={{ fontSize: '0.85em', fontWeight: 'normal', color: '#888', marginLeft: 8 }}>Respect the Legacy</span>

혁신은 낯선 경험이 아닙니다. 완전히 새로운 학습을 강요하기보다, 검증된 유산 위에서 점진적으로 개선합니다.

### 2. 사용자가 쉽고 빠르게 예측가능한 제품을 만듭니다. <span style={{ fontSize: '0.85em', fontWeight: 'normal', color: '#888', marginLeft: 8 }}>Predictability</span>

사용자를 고민하게 만들지 않습니다. 사용자는 생각하지 않아도 다음 행동을 예측할 수 있어야 합니다. 일관된 레이아웃과 피드백을 통해 신뢰를 설계합니다.

### 3. 사용자가 고민하지 않고 즉시 행동할 수 있도록 유도합니다. <span style={{ fontSize: '0.85em', fontWeight: 'normal', color: '#888', marginLeft: 8 }}>Affordance</span>

생각하지 않아도 무엇을 해야 할지 직관적으로 알게 합니다. 설명서 없이도 자연스럽게 행동을 유도하는 디자인을 추구합니다.

### 4. 사용자에게 일관된 제품 경험을 제공합니다. <span style={{ fontSize: '0.85em', fontWeight: 'normal', color: '#888', marginLeft: 8 }}>Consistency</span>

아임인의 어떤 서비스를 이용하더라도 사용자는 동일한 '아임인스러움'을 느껴야 합니다. 좋은 UI는 설명하지 않습니다. 다음 행동이 자연스럽고, 누르기 쉽고, 실수하기 어려운 경험을 만듭니다.
